1 前言

最近了解到hexo支持的博客系统很不错，界面简介大方，是众多程序员的标配，所以就想上手一个。听小伙伴说大概一个小时就能搞定，而我却画了整整一天的时间（包括设置主题）。中间主要因为一些bug浪费了一些时间，这也是程序员小白经常遇到的问题。所以，在我的第一篇博客中，就把这些贴出来，大家一起讨论学习。

2 hexo安装配置

对于hexo的基本配置，在网上都能搜到很多详细的过程，对于hexo的安装，建议大家参考官方文档。当然对于有些省略的步骤，比如node.js的安装，git的安装可以查看其他教程。步骤都比较简单，轻而易举就能完成。

3 github设置

就是对于github的使用，在这里建议大家可以简单了解，因为我们最后的博客是要在github page上发布的，而每个github用户只能发布一个页面。因此，对于github的使用了解的不多，就容易掉进坑里面，我昨天就遇到了的这样的问题。

首先，我们先了解一下github。

gitHub是一个面向开源及私有软件项目的托管平台，因为只支持git 作为唯一的版本库格式进行托管，故名gitHub。

这里大家看到了：

  • github可以托管我们的代码，减少本地资源丢失所造成的损失。其次，我们可以才github上找到开源的项目，这可供我们学习交流。总而言之，github是一个 平台。
  • 那要在这个平台活动，就要有相应的 工具，这里我们要用到Git ，Git是一款免费，开源的分布式版本控制系统。至于具体怎么理解，大家感兴趣可以移步度娘。

github page给用户提供了发布个人主页的平台。我们将可以将自己的个人主页托管在这个平台上。目前只能托管静态内容。因此，我们可以将hexo项目托管到该平台上。

hexo, git, github 的关系已经很清除了，hexo提供了一个博客网站框架，可以渲染我们的内容，变成精美的页面。git和github将我们的网站托管，这样就可以被广大网友访问到。
