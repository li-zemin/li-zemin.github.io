机器学习系列(13)——主成分分析

发表于 2017-10-23 | 分类于 机器学习 | | 阅读次数:

​ 讨论因子分析时，我们给出了一个对“大概”来源于k个子空间的数据\(x \in \mathbb{R}\)，k << n。特别地，我们设想每个数据点\(x ^{(i)}\)首先由k维的放射空间\(\left \{ \Lambda z + \mu; z \in \mathbb{R} ^{k} \right \}\)生成，再加上一些方差为ψ的噪声。因子分析基于一个概率模型，同时参数估计使用了迭代的EM算法。

​ 在本节内容里，我们将研究一种新方法，主成分分析，Principla Components Analysis。这种方法试图识别这些数据来源的子空间。只需要一个特征向量计算而不需要借助EM算法，PCA可以很直接地完成这样的工作。

​ 假设我们给定一个数据集\(\left \{ x ^{(i)};i=1,…,m \right \}\)是m个不同种类的汽车，它们的最大速度，回转半径等参数有所差异。对于每个i，设\(x ^{(i)} \in \mathbb{R}\)（n << m）。但是有些我们是不知道的，比如两个不同的属性\(x _i\)和\(x _j\)可能满足一种依赖的线性关系，比如一辆汽车每小时跑多少英里，与每小时跑多少千米，因此，这两个属性其实表达了一种信息。那么，这些数据就可以说是来源于n-1维的子空间。我们如何能够自动检测，同时去掉一些冗余的属性特征呢?

​ 介绍一个实际的例子，考虑一个关于遥控直升机操作手的数据集，\(x _1 ^{(i)}\)表示操作手i的操作技能，\(x _2 ^{(i)}\)表示操作手i对飞行的喜爱程度。因为操作遥控直升机飞行是很困难的，只有经常投入的，真正喜爱飞行的，可以称为好的操作手。因此，这两个属性\(x _1\)和\(x _2\)之间具有很强的联系。我们可以假设这些数据沿着二维坐标的对角线分布，\(\mu _{1}\)方向表示了一个人的飞行潜质，有很少的噪声点不在这个轴上。那么，我们 如何自动计算得到\(\mu _{1}\)的方向呢？
